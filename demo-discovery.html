This demonstrates how the elicitation questions work through MCP tools

Note: The elicitation questions don't appear automatically in the UI. Instead, they are accessed through MCP tools that a client application (like Claude Desktop) would call.

Step 1: Start Discovery

MCP Tool Call: start_form_discovery
First Question:
"What type of change do you want to make to your insurance policy?"
Options: Change beneficiary, Get a loan, Surrender policy, Reinstate policy, Change annuity contract, Request amendment, Other

Step 2: Answer Questions

MCP Tool Call: answer_discovery_question { answer: "Change beneficiary" }
User Answer: "Change beneficiary"
Next Question:
"Do you need to make this change immediately, or are you planning ahead?"
Options: Immediately, Planning ahead, Not sure

Step 3: Continue Discovery

MCP Tool Call: answer_discovery_question { answer: "Immediately" }
User Answer: "Immediately"
Next Question:
"What is your current situation?"
Options: Life event (marriage, divorce, birth), Financial planning, Estate planning, Other

Step 4: Get Results

MCP Tool Call: get_discovery_results
Recommended Forms:
1. Beneficiary Change Request (beneficiary-change)
Description: Form to update the beneficiary designation on your life insurance policy

Available MCP Tools

  • start_form_discovery - Begin the discovery process
  • get_discovery_question - Get current question
  • answer_discovery_question - Provide an answer
  • get_discovery_results - Get recommended forms
  • reset_discovery - Start over
  • get_form_summary - Get details about a specific form
  • get_form_pdf - Download the form PDF
How to Test:
1. Connect Claude Desktop to this MCP server
2. Ask Claude to help you find a form
3. Claude will use these tools to guide you through the discovery process
4. You'll see the questions and can provide answers naturally
